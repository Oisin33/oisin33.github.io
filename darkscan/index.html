Detects manipulative “dark pattern” UI — confirmshaming, fake urgency, hidden costs, sneaky opt-outs and more — in any page text. It pairs fast keyword rules with a trained classifier that generalises to wording it has never seen. Everything below runs entirely client-side: the real ML model is exported to JSON and the inference happens in JavaScript. No server, no data leaves your machine.

How it works

Dark patterns are deceptive interface choices that nudge you into decisions you wouldn’t otherwise make. Pure keyword matching catches the obvious ones but misses anything reworded — so DarkScan runs a cascade of detectors, cheapest first:

1 · Heuristics
regex / keyword rules — instant, precise on known phrasings
2 · Classifier
TF-IDF + logistic regression — generalises to unseen wording
3 · Embeddings*
semantic match — catches different vocabulary entirely
4 · LLM*
adjudicates the genuinely ambiguous cases

This demo runs tiers 1 and 2 (tiers 3–4 are server-side options in the full project, marked *). The classifier here is the actual trained scikit-learn model — its word + character n-gram vocabulary, IDF weights and regression coefficients are exported to model.json and the maths is re-implemented in detector.js, verified to match the Python model to within 0.0001.

Character n-grams are what let it generalise: “pay full price like a chump” shares sub-word features with the training example “pay full price”, so it’s flagged as confirmshaming even though that exact sentence was never in the data.

Under the hood — the full system

The browser demo above is one slice. The actual project is a full-stack application: a browser extension that scans live pages, a REST API that serves and persists results, a React analytics dashboard, and the Python detection engine with its learning loop. Here's how the pieces fit.

Browser extension
Manifest V3

One-click scan of the current tab. A content script extracts the page HTML and posts it to the API; a popup renders the findings in real time.

REST API
Flask · SQLite

Endpoints for scanning, history, stats, and the learning loop (/feedback, /distilled, /retrain). Scans persist to SQLite with a connection-per-request model.

Dashboard
React · Recharts

Scan history, severity trends, per-domain breakdowns, and a Review & Train console where you confirm/reject detections and retrain the model.

Detection engine
Python · scikit-learn

The heuristic rules plus the tiered ML cascade. Segments a page into UI spans, classifies each, and merges results — heuristics for known patterns, ML for novel ones.

Engineering decisions & problems solved

  • A cost-aware cascade, not a single model. Cheap tiers settle the easy cases; only genuinely ambiguous spans escalate to embeddings, then an LLM. A 200-span page costs a handful of LLM calls instead of 200.
  • A gated distillation loop. When the expensive tiers confidently resolve something the cheap one couldn't, that verdict is captured to teach the cheap tier — but it's gated (high-confidence only), kept in a separate store, excluded from training by default, and always overridable by human feedback. Auto-training on a model's own guesses causes drift; the gating is the point.
  • A real false-positive, fixed. The original "disguised ads" rule matched the substring brand, so Bootstrap's ubiquitous navbar-brand flagged a huge share of normal sites. Narrowed to word-boundary ad tokens; regression-tested.
  • Label validation on the feedback loop. User corrections are checked against the taxonomy server-side, so a typo can't create a phantom class that poisons the next retrain.
  • Tested. 19 backend tests cover the retriever, the cascade routing, the distillation guardrails, and the detection logic — all runnable with no API key.
