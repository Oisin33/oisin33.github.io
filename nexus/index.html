How it works

When you ask a question, the agent runs a tool-use loop:

  • search_codebase — BM25 over indexed chunks
  • read_file — pull a full file
  • list_files — explore structure
  • generate_docs — summarise a module

It calls tools, reads results, and decides what to do next — up to a safety cap on rounds — then streams a final answer over SSE with the source files it consulted.

Under the hood

Nexus is a small distributed system, not a single script. The interesting engineering is in the retrieval, the agent loop, and the streaming — and in the bugs that only show up when you actually run it against real repos.

Backend
FastAPI · async

Indexes a repo by fetching files concurrently (asyncio.gather with a semaphore), chunks them, and builds a BM25 index. Streams responses over Server-Sent Events.

Agent loop
Claude · tool-use

A multi-round tool-calling loop with four tools and a safety cap on rounds. Tracks which files it consulted and returns them as sources.

Front end
TypeScript · React

Live token streaming, a file tree that highlights consulted files, and a syntax-highlighted viewer — strict tsc build, no any escapes.

Infra
Docker · nginx

Compose stack with an nginx reverse proxy tuned for SSE (long read timeouts, buffering off), the backend on an internal network, and a CI pipeline.

Bugs found & fixed (the real work)

  • BM25 returned nothing for common queries. The scorer produces negative scores when a term appears in many documents, and the code cut results off at zero — so a query like "how does auth work" silently returned no results. Fixed by gating on token overlap instead of score sign; regression-tested.
  • The per-file cap fought the result limit. Capping at N before applying a per-file diversity limit meant one large file could starve the results. Reordered so other files fill the gaps.
  • Fake streaming → real streaming. An early version chunked a finished response to look live; replaced with genuine token streaming from the model over SSE, and fixed an nginx timeout that silently killed long responses mid-answer.
  • Container networking. The frontend hardcoded localhost:8000, which breaks inside Docker; switched to a relative path proxied by nginx so dev and container both work.
  • Tested. The agent loop is covered by tests using a mocked Anthropic client — tool dispatch, the round cap, and source tracking all run in CI with no API key.

The full audit trail, with diffs and test output, is in the repository.
